# 问题记录：科幻管道拖尾特效消失太慢

## 问题描述
在选择“科幻管道”（Sci-Fi Pipeline）拖尾特效时，鼠标停止移动后，特效仍会持续较长时间（超过 600ms）才完全消失。这会导致在进行文字输入或查看文字时，残留的特效干扰视线。

对比其他拖尾特效的持续时间：
- **普通线段 (`Line`)**: 约 300ms
- **电弧 (`Electric`)**: 约 400ms
- **流光 (`Streamer`)**: 约 500ms

## 解决方案
为了在不牺牲拖尾长度（视觉质量）的前提下压缩消失时间，采取了以下优化方案：

### 1. 提升物理收缩速度
修改 `TubesRenderer` 构造函数中的 `lag` 参数。`lag` 值越高，节点跟随前一节点的速度越快。对比之前：
- 管道 0：`0.25f` (原始) -> `0.40f` (过快) -> `0.30f` (最终优化)
- 管道 1：`0.35f` (原始) -> `0.50f` (过快) -> `0.40f` (最终优化)
- 管道 2：`0.45f` (原始) -> `0.60f` (过快) -> `0.50f` (最终优化)

### 2. 引入停顿检测（Idle Detection）
原逻辑仅在 `points` 列表彻底清空（通常需要 350ms）后才开始检查收敛。
新逻辑通过 `points.back().addedTime` 检测鼠标状态：
- 若鼠标停顿超过 **50ms**，立即判定为 `isIdle` 状态并触发渐隐逻辑。

### 3. 加快渐隐速度
- 将渐隐时的 alpha 递减量从 `25.0` 提升至 `35.0`。
- 将收敛判定阈值从 `5px` 提升至 `10px`，使特效能更早进入快速消失阶段。

## 最终效果
优化后，特效在鼠标停止后约 **400ms** 内即可完全消失，与“电弧”特效时长相当，显著减少了对文字显示的干扰。
