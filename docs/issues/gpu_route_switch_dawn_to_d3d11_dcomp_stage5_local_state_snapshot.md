# GPU Route Switch Stage 5: Unified Local State Snapshot

Date: 2026-02-12

## Goal
Ensure each settings-state fetch writes a fresh local diagnostics file that Codex can read directly after user test runs.

## Changes
- `WebSettingsServer::BuildStateJson()` now persists the returned state JSON to:
  - `<exe_dir>/.local/diag/web_state_auto.json`
- Added automatic directory creation for `.local/diag` if missing.
- Snapshot is overwritten on each state build (`truncate`), keeping only the latest state.

## Why
Recent test rounds showed stale diagnostics files from older runtime paths. This change establishes a deterministic single-source state snapshot generated by the current running binary.

## Validation
- Build target: `Release|x64`
- Runtime check: after opening settings page, `x64/Release/.local/diag/web_state_auto.json` timestamp should refresh.

## Risk
- Low. Read-only snapshot write path; no rendering behavior changes.
